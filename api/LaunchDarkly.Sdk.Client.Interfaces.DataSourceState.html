BackgroundDisabled

Indicates that the SDK is in background mode and background updating has been disabled.

Declaration
public const DataSourceState BackgroundDisabled
Field Value
Type Description
DataSourceState
Remarks

On mobile devices, if the application containing the SDK is put into the background, by default the SDK will still check for feature flag updates occasionally. However, if this has been disabled with EnableBackgroundUpdating(false), the SDK will instead stop the data source and wait until it is in the foreground again. During that time, the state is BackgroundDisabled.

Interrupted

Indicates that the data source encountered an error that it will attempt to recover from.

Declaration
public const DataSourceState Interrupted
Field Value
Type Description
DataSourceState
Remarks

In streaming mode, this means that the stream connection failed, or had to be dropped due to some other error, and will be retried after a backoff delay. In polling mode, it means that the last poll request failed, and a new poll request will be made after the configured polling interval.

This is different from NetworkUnavailable, which would mean that the SDK knows the device is not online at all and is waiting for it to be online again.

NetworkUnavailable

Indicates that the SDK is aware of a lack of network connectivity.

Declaration
public const DataSourceState NetworkUnavailable
Field Value
Type Description
DataSourceState
Remarks

On mobile devices, if wi-fi is turned off or there is no wi-fi connection and cellular data is unavailable, the device OS will tell the SDK that the network is unavailable. The SDK then enters this state, where it will not try to make any network connections since they would be guaranteed to fail, until the OS informs it that the network is available again.

This is different from Interrupted, which would mean that the SDK thinks network requests ought to be working but for some reason they are not (due to either a network problem that the device OS does not know about, or a problem with the service endpoint).

The .NET Standard version of the SDK is not able to detect network status, so desktop applications will see the Interrupted state if the network is turned off.

Type Description
DataSourceState
Remarks

This means that either the SDK was originally configured with Offline(true) and has not been changed to be online since then, or that it was originally online but has been put offline with SetOffline(Boolean, TimeSpan) or SetOfflineAsync(Boolean). It is not a permanent condition; the application can change this at any time.
