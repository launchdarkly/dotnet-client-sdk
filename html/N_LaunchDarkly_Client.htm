LaunchDarkly.Client Namespace

This namespace contains types that are shared between the Xamarin and .NET SDKs, such as User.

CustomEvent
An analytics event generated by the Track method. Used internally.
EvaluationDetailT
An object returned by the "variation detail" methods of the client, combining the result of a flag evaluation with an explanation of how it was calculated.
EvaluationReason
Describes the reason that a flag evaluation produced a particular value. Subclasses of EvaluationReason describe specific reasons.
EvaluationReasonError Obsolete.
Indicates that the flag could not be evaluated, e.g. because it does not exist or due to an unexpected error. In this case the result value will be the default value that the caller passed to the client.
EvaluationReasonFallthrough Obsolete.
Indicates that the flag was on but the user did not match any targets or rules.
EvaluationReasonOff Obsolete.
Indicates that the flag was off and therefore returned its configured off value.
EvaluationReasonPrerequisiteFailed Obsolete.
Indicates that the flag was considered off because it had at least one prerequisite flag that either was off or did not return the desired variation.
EvaluationReasonRuleMatch Obsolete.
Indicates that the flag was considered off because it had at least one prerequisite flag that either was off or did not return the desired variation.
EvaluationReasonTargetMatch Obsolete.
Indicates that the user key was specifically targeted for this flag.
