Tells the client that all pending analytics events should be delivered as soon as possible.

Remarks

When the LaunchDarkly client generates analytics events (from flag evaluations, or from Identify(User, TimeSpan) or Track(String)), they are queued on a worker thread. The event thread normally sends all queued events to LaunchDarkly at regular intervals, controlled by the EventFlushInterval(TimeSpan) option. Calling Flush triggers a send without waiting for the next interval.

Flushing is asynchronous, so this method will return before it is complete. However, if you shut down the client with Dispose, events are guaranteed to be sent before that method returns.
