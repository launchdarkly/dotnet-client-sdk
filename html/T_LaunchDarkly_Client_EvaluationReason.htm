Remarks
Note that this is currently a base class with subclasses corresponding to each EvaluationReasonKind. In a future version of the SDK, it will instead be a value type (struct). Therefore, instead of checking for the subclasses, use the Kind property to distinguish between reasons, and instead of constructing the subclasses directly, use factory methods/properties such as RuleMatchReason(Int32, String) or FallthroughReason.
