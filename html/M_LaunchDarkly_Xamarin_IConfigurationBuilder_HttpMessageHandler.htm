Remarks
This is exposed mainly for testing purposes; you should not normally need to change it. The default value is an HttpClientHandler, but if you do not change this value, on mobile platforms it will be replaced by the appropriate native HTTP handler for the current current platform, if any (e.g. Xamarin.Android.Net.AndroidClientHandler). If you set it explicitly to , the SDK will call the default HttpClient constructor without specifying a handler, which may or may not result in using a native HTTP handler (depending on your application configuration).
